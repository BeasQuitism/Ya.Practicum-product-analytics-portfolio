### Анализ данных бизнеса в ClickHouse (Сервис чтения и прослушивания книг)

**Задача:** Провести комплексный анализ пользовательского поведения, потребления контента и технического состояния мобильных приложений в крупном сервисе для чтения и прослушивания книг с использованием аналитической СУБД ClickHouse.

**Что сделал:**
- Написал сложные SQL-запросы в ClickHouse с использованием продвинутых функций и конструкций: `argMax`, `sumIf`, `arrayExists`, `multiIf`, CTE (`WITH`) и работа с массивами.
- **Сегментация аудитории:** Разделил пользователей на "Читателей", "Слушателей" и "Универсалов" на основе анализа паттернов потребления контента (соотношение времени чтения и прослушивания к общему времени).
- **Кросс-платформенный анализ:** Определил основную платформу (iOS или Android) для каждого пользователя с помощью функции `argMax`, чтобы понять, с какого устройства пользователь потребляет контент чаще.
- **Аудит версий приложений:** Вычислил последнюю активную версию приложения для каждого пользователя и сравнил её с актуальной версией на платформе, рассчитав долю пользователей на устаревших версиях.
- **Контент-аналитика:** Рассчитал метрики вовлеченности (топ-20 городов, топ-5 книг, топ-10 авторов по суммарному количеству часов прослушивания/чтения).
- **Проверка качества данных:** Выполнил аудит корректности контент-тегов (на примере категории "Магия"), выявив книги с некорректной или отсутствующей разметкой в каталоге.

**Результаты:**
- Подготовил аналитическую базу для маркетинга и продуктовой команды: сегментация пользователей и анализ платформ позволяют точечно управлять рекламными активностями и push-уведомлениями.
- Выявил долю пользователей, использующих устаревшие версии приложений, что дает сигнал для команды разработки о необходимости внедрения принудительных обновлений или мотивирующих пушей.
- Обнаружил аномалии в контент-каталоге (книги с "магией" в названии без соответствующего тега), что помогает команде контента улучшить навигацию, рекомендации и поиск для пользователей.

**Инструменты:** ClickHouse, SQL (продвинутый уровень: CTE, специфичные агрегатные функции, работа с массивами), Jupyter Notebook.
