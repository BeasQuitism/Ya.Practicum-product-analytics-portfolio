## A/B-анализ и поиск инсайтов в финтех-приложении (расширенная версия)

**Задача:** Провести полный цикл A/B-тестирования новой фичи онбординга в финтех-приложении, выявить узкие места в воронке конверсии и оценить влияние на ключевые бизнес-метрики

**Что сделал:**
- Провёл предобработку исторических данных о 54,805 пользователях (период: 2 июня – 15 июня 2025)
- Построил воронку конверсии из 7 шагов (от install до second_deposit) и выявил узкие места:
  - **Критический отток:** onboarding_complete → first_deposit (теряется 19.4% пользователей)
  - **Второй узкий место:** first_deposit → asset_purchase (теряется 44.4% пользователей)
  - **Третий узкий место:** asset_purchase → second_deposit (теряется 64.9% пользователей)
- Проанализировал конверсию по уровням риска пользователей (low/medium/high) и выявил, что пользователи с высоким риском имеют конверсию на 15% ниже
- Применил статистические тесты для оценки значимости результатов A/B-теста:
  - **t-test** для сравнения средних значений
  - **Mann-Whitney U test** для сравнения распределений
  - **Bootstrap-анализ** для построения 95% доверительных интервалов
- Проверил 4 метрики: ключевую (средняя сумма депозита на пользователя), барьерную (конверсия в 1-й депозит) и 2 вспомогательные метрики
- Построил визуализации: графики накопительного p-value, распределения метрик, сравнение групп

**Результаты:**
- ✅ **Ключевая метрика выросла на +2.45%** (статистически значимо, p < 0.0001)
- ✅ **Конверсия из 1-го во 2-й депозит увеличилась на +12.35 п.п.** (с 20.26% до 32.61%, p < 0.0001)
- ✅ **Средняя сумма на платящего пользователя выросла на +4.35%** (статистически значимо)
- Выявил, что новая фича онбординга наиболее эффективна для пользователей с низким и средним уровнем риска
- Сформулировал рекомендации по дальнейшей оптимизации воронки:
  - Упростить процесс первой покупки (asset_purchase)
  - Добавить триггерные уведомления для возврата пользователей после первого депозита
  - Провести дополнительный A/B-тест для сегмента high-risk пользователей
- Дала рекомендацию по раскатке фичи на всех пользователей с учетом сегментации

**Инструменты:** Python (pandas, numpy, scipy, statsmodels, matplotlib, seaborn), SQL, Jupyter Notebook
