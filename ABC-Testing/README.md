### A/B/C-тестирование в сервисе заказа поездок (продвинутый уровень)

**Задача:** Проверить продуктовую гипотезу о том, что новая математическая модель прогнозирования времени ожидания повысит качество пользовательского опыта и увеличит конверсию из создания заказа в успешную поездку в сервисе заказа поездок (аналог Яндекс Go)

**Что сделал:**
- **Дизайн эксперимента:** Спроектировал A/B/C-тест с тремя группами (control, treatment_1, treatment_2) с равномерным сплитом трафика 33%/33%/33%
- **Расчёт MDE (Minimum Detectable Effect):** Рассчитал минимальный детектируемый эффект на разных уровнях трафика (от 10% до 100%), определил оптимальную долю трафика для теста с учётом временных ограничений (2 недели) и требуемого MDE ≤ 4%
- **Линеаризация метрик:** Применил метод линеаризации для ratio-метрик (метрики-отношения), что позволило корректно сравнивать группы и снизить дисперсию
- **Кумулятивный анализ:** Рассчитал и визуализировал кумулятивные значения ключевой метрики (конверсия в успешную поездку) и барьерной метрики (выручка на пользователя) для отслеживания динамики во времени и выявления искажений пользовательского поведения
- **Множественные сравнения:** Применил поправку Бонферрони для корректной проверки трёх гипотез одновременно (control vs treatment_1, control vs treatment_2, treatment_1 vs treatment_2)
- **Мониторинг вспомогательных метрик:** Отслеживал конверсию из просмотра формы в создание заказа и количество поездок на пользователя для комплексной оценки эффекта
- **Очистка данных:** Обработал пользователей, попавших в несколько групп одновременно, исключил заказы со статусом cancelled_by_driver

**Результаты:**
- ✅ **Группа treatment_1** показала статистически значимый рост ключевой метрики (p-value < 0.05 с учётом поправки Бонферрони)
- ✅ **Группа treatment_2** не показала статистически значимых различий с контрольной группой
- ✅ Барьерная метрика (выручка на пользователя) не снизилась в тестовых группах
- ✅ Вспомогательные метрики подтвердили положительный эффект новой модели
- **Итоговая рекомендация:** Внедрить фичу для всех пользователей в варианте treatment_1

**Инструменты:** Python (pandas, scipy.stats, matplotlib), статистические тесты (t-test), линеаризация ratio-метрик, поправка Бонферрони, кумулятивный анализ
